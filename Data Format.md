# Synopsys 常见文件格式说明

- **`*.vg`, `*.g.v`** – Verilog gate-level netlist file.  
  Sometimes people use these file extensions to differentiate source files and gate-level netlists.

- **`*.svf`** – Automated Setup File.  
  This file helps Formality process design changes caused by other tools used in the design flow. Formality uses this file to assist the compare-point matching and verification process. This information facilitates alignment of compare points in the designs that you are verifying. For each automated setup file that you load, Formality processes the content and stores the information for use during the name-based compare point matching period.

- **`*.ddc`** – Synopsys internal database format.  
  This format is recommended by Synopsys to store gate-level netlists.

- **`*.vcd`** – Value Change Dump format.  
  This format is used to save signal transition trace information. Since it is a text format, VCD files can become very large. Synopsys provides tools such as `vcd2vpd`, `vpd2vcd`, and `vcd2saif` to convert between formats.

- **`*.vpd`** – VCD Plus.  
  A proprietary compressed binary trace format from Synopsys used to save signal transition trace information.

- **`*.saif`** – Switching Activity Interchange Format.  
  Another format for storing signal transition activity. SAIF supports signals, ports, generates, enumerated types, records, arrays, and integers.

- **`*.tcl`** – Tool Command Language script.  
  Tcl is the scripting language used to drive Synopsys tools.

- **`*.sdc`** – Synopsys Design Constraints.  
  SDC is a Tcl-based constraint format used to communicate design intent between EDA tools. It typically contains:
  - SDC version
  - Units
  - Timing constraints
  - Design constraints
  - Comments

- **`*.lib`** – Technology Library source file.  
  Liberty libraries contain characterization data for standard cells, including:
  - Cell names
  - Pin names
  - Area
  - Delay
  - Pin capacitance
  - Design rule constraints
  - Operating conditions
  - Wire-load models

- **`*.db`** – Technology Library.  
  Compiled binary version of the `.lib` file.

- **`*.plib`** – Physical Library source file.  
  Contains physical layout information required for:
  - Floorplanning
  - RC estimation
  - Placement
  - Routing

- **`*.pdb`** – Physical Library.  
  Compiled version of the `.plib` physical library.

- **`*.slib`** – Symbol Library source file.  
  Defines graphical symbols for schematic representation of library cells. Used by Design Compiler and Design Vision.

- **`*.sdb`** – Symbol Library.  
  Compiled version of the `.slib` symbol library.

- **`*.sldb`** – DesignWare Library.  
  Contains DesignWare IP library information.

- **`*.def`** – Design Exchange Format.  
  Commonly used by Cadence tools to represent physical layout. Synopsys traditionally used Milkyway instead.

- **`*.lef`** – Library Exchange Format.  
  Describes standard-cell physical abstracts. Widely used in Cadence tools; Synopsys historically converted LEF into Milkyway/NDM.

- **`*.rpt`** – Report file.  
  Plain-text reports generated by Synopsys tools.

- **`*.tf`** – Vendor Technology File.  
  Contains technology-specific information such as:
  - Metal layer definitions
  - Physical/electrical characteristics
  - Design rules

- **`*.itf`** – Interconnect Technology File.  
  Describes process cross-section and connectivity, including conductor and dielectric layer properties.

- **`*.map`** – Mapping file.  
  Maps names used in the vendor technology file to those used in the process ITF.

- **`*.tluplus`** – TLUPlus file.  
  Generated from ITF files. Contains advanced RC extraction models used by Synopsys place-and-route tools.

- **`*.spef`** – Standard Parasitic Exchange Format.  
  Standard format used to store parasitic resistance and capacitance extracted after place-and-route.

- **`*.sbpf`** – Synopsys Binary Parasitic Format.  
  Proprietary compressed binary version of SPEF for faster loading and smaller file size.

- **`*.nmw` / Milkyway Database** – Synopsys physical design database.  
  Contains:
  - Standard cell libraries
  - Macro libraries
  - Physical layout
  - Timing information
  - Floorplan
  - Placement
  - Routing
  - Design hierarchy

  Milkyway libraries are divided into several views, such as:
  - CEL (cell view)
  - FRAM (frame view)
  - NET
  - PIN
  - etc.

  > **Note:** In ICC2, Milkyway has been replaced by **NDM (New Data Model)**.

- **`simv`** – Compiled simulation executable.  
  Generated by Synopsys VCS after compilation.

- **`alib-52`** – Characterized target technology library cache.  
  A pseudo-library generated by Design Compiler that accelerates optimization by caching technology characterization data.
